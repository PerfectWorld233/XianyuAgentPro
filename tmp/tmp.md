我看过这个仓库了。它目前本质上是一个：

  “闲鱼消息自动回复 Agent”基础框架

  核心结构大概是：

  - main.py：主流程、WebSocket 长连接、收消息/发消息、心跳、token 刷新
  - XianyuApis.py：闲鱼相关 HTTP API 封装、cookie/token 获取
  - XianyuAgent.py：意图识别 + LLM 回复生成
  - context_manager.py：SQLite 会话上下文、商品信息缓存、议价次数
  - prompts/：分类、议价、技术、默认回复的 prompt

  所以它天然更偏向：
  “自动回复引擎”，而不是完整的**“客服产品”**。
